# S3-Compatible Backups Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add automated full-system backups to any S3-compatible storage (AWS S3, Minio, R2, B2). Backups include all app directories and both SQLite databases as a single tar.gz archive. Admins configure frequency/retention through the UI and can trigger manual backups.

**Architecture:** New `backup_config` and `backup_history` tables in audit.db (singleton config + history). `S3Config` parsed from env vars at startup. A background tokio task sleeps until next scheduled backup, waking on manual trigger via mpsc channel or shutdown via CancellationToken. Archive creation uses `VACUUM INTO` for DB snapshots and tar+flate2 for the archive. S3 operations via `rust-s3` crate. An `AtomicBool` on AppState prevents concurrent backups.

**Tech Stack:** Rust, Axum, sqlx (SQLite), minijinja, htmx, rust-s3, tar, flate2, tokio (mpsc + CancellationToken), metrics

**Spec:** `docs/superpowers/specs/2026-03-30-s3-backups-design.md`

**Files:**

- [ ] **Step 1:** Add private S3 helper functions in `src/backup.rs`:

  - `async fn list_backups(bucket: &s3::Bucket) -> eyre::Result<Vec<String>>` -- LIST objects with prefix `backups/`, return sorted keys.

  - `async fn enforce_retention(bucket: &s3::Bucket, retention_count: i64) -> eyre::Result<usize>` -- list backups, sort by key, delete oldest beyond retention_count. Return count deleted.

- [ ] **Step 2:** Add `pub async fn run_backup(state: &AppState, s3_config: &S3Config, trigger_source: &str)`:
  - Acquire lock via `AtomicBool::compare_exchange` on `state.backup_running`. If already true, return early (log warning for scheduled, error for manual).
  - Create `BackupLockGuard` to ensure release on any exit path.
  - Insert backup_history row via `state.audit.start_backup_record(trigger_source)`.
  - Log audit event `backup_started`.
  - Call `create_archive(&state.config.data_dir, &state.pool, state.audit.pool_ref())`.
  - Check archive size -- fail if >4GB (4_294_967_296 bytes).
  - Create bucket handle via `create_bucket(s3_config)`.
  - Generate S3 key: `format!("backups/{}.tar.gz", chrono::Utc::now().format("%Y-%m-%dT%H-%M-%SZ"))`.
  - Upload via `upload_archive(&bucket, &key, &archive_path)`.
  - On upload failure: call `state.audit.fail_backup_record(...)`, log audit `backup_failed`, return.
  - On success: call `state.audit.complete_backup_record(...)`, log audit `backup_completed`.
  - Read `retention_count` from backup config. Call `enforce_retention(&bucket, retention_count)`. Log warnings on failure but do not fail the backup.
  - Prune backup history: `state.audit.prune_backup_history(50)`.
  - Clean up archive temp file.
  - Update Prometheus metrics (see Task 9).

- [ ] **Step 3:** Add `pub fn calculate_next_backup_delay(last_success: Option<&BackupRecord>, frequency_hours: i64) -> std::time::Duration`:
  - If no last_success, return Duration::ZERO (backup immediately).
  - Parse last_success.started_at with chrono, compute next = last + frequency_hours. If next <= now, return Duration::ZERO. Otherwise return next - now as Duration.

- [ ] **Step 4:** Add `pub fn spawn_backup_task(state: AppState, s3_config: S3Config, mut trigger_rx: mpsc::Receiver<()>, cancel_token: CancellationToken)`:
  - Spawns a tokio task with a loop.
  - At top of loop: read backup_config from DB. If DB read fails, log error, sleep 60s (with select on cancel), continue.
  - On startup (first iteration only), check for stuck backups: query latest_backup, if is_backup_stuck, call fail_backup_record with "Server restarted during backup", reset AtomicBool.
  - If enabled=false: enter two-way select on trigger_rx.recv() and cancel_token.cancelled(). On trigger: run_backup with "manual". On cancel: return.
  - If enabled=true: compute delay via calculate_next_backup_delay. Enter three-way select:
    - `tokio::time::sleep(delay)` -> run_backup with "scheduled"
    - `trigger_rx.recv()` -> run_backup with "manual"
    - `cancel_token.cancelled()` -> return
  - After run_backup completes (success or failure), loop continues.

- [ ] **Step 5:** In `src/main.rs`, replace the TODO comment from Task 4 with the actual spawn. Note: `backup_trigger_rx` (`Option<Receiver<()>>`) is consumed by the spawn, and `backup_cancel` (`Option<CancellationToken>`) is borrowed:
  ```rust
  if let (Some(s3_cfg), Some(rx), Some(ref cancel)) =
      (state.s3_config.clone(), backup_trigger_rx, &backup_cancel)
  {
      substrukt::backup::spawn_backup_task(
          state.clone(),
          s3_cfg,
          rx,
          cancel.child_token(),
      );
  }
  ```
  `state.s3_config.clone()` is needed because we need an owned `S3Config` to pass to the spawned task, and `state.s3_config` is behind `Arc`. `backup_trigger_rx` moves into the `if let` and then into the spawn. `backup_cancel` is referenced after the spawn (for shutdown cancellation), so it's borrowed here.
